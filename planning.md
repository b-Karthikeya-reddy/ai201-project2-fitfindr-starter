## Tools

### Tool 1: search_listings

**What it does:**
Searches the listings dataset for secondhand clothing items that match the user's description, size, and maximum price.

**Input parameters:**

- `description` (str): Keywords describing the desired item.
- `size` (str): User's preferred clothing size.
- `max_price` (float): Maximum amount the user is willing to spend.

**What it returns:**
A list of matching listing dictionaries containing information such as title, description, size, condition, price, brand, colors, platform, and style tags.

**What happens if it fails or returns nothing:**
Returns an empty list. The agent informs the user that no matches were found and suggests changing the description, size, or price range.

---

### Tool 2: suggest_outfit

**What it does:**
Generates outfit suggestions by combining the selected listing item with pieces already present in the user's wardrobe.

**Input parameters:**

- `new_item` (dict): The selected listing returned from search_listings.
- `wardrobe` (dict): The user's wardrobe information.

**What it returns:**
A string containing one or more outfit recommendations and styling suggestions.

**What happens if it fails or returns nothing:**
If the wardrobe is empty, the tool returns general styling advice using the selected item instead of specific wardrobe recommendations.

---

### Tool 3: create_fit_card

**What it does:**
Creates a short social-media style caption describing the outfit and featured item.

**Input parameters:**

- `outfit` (str): Outfit recommendation generated by suggest_outfit.
- `new_item` (dict): The selected listing item.

**What it returns:**
A short shareable caption suitable for Instagram or social media.

**What happens if it fails or returns nothing:**
Returns an informative error message explaining that the outfit information is incomplete and a fit card cannot be generated.

---

## Planning Loop

The agent first calls search_listings using the user's query information. If no listings are found, the agent stores an error message and stops. If listings are found, the first matching item is selected and stored in session state. The agent then calls suggest_outfit using the selected item and wardrobe information. After an outfit suggestion is generated, create_fit_card is called using the outfit suggestion and selected item. The final session containing the selected item, outfit suggestion, and fit card is returned to the user.

---

## State Management

The agent uses a session dictionary to store information between tool calls. After search_listings completes, the selected item is stored as session["selected_item"]. After suggest_outfit completes, the generated recommendation is stored as session["outfit_suggestion"]. After create_fit_card completes, the generated caption is stored as session["fit_card"]. Each tool uses information saved by previous tools instead of asking the user for the same information again.

---

## Error Handling

| Tool            | Failure mode                          | Agent response                                                                   |
| --------------- | ------------------------------------- | -------------------------------------------------------------------------------- |
| search_listings | No results match the query            | Inform the user that no matching items were found and suggest adjusting filters. |
| suggest_outfit  | Wardrobe is empty                     | Generate general styling advice instead of wardrobe-specific recommendations.    |
| create_fit_card | Outfit input is missing or incomplete | Return an informative error message and do not generate a fit card.              |

## A Complete Interaction (Step by Step)

**Example user query:** "I'm looking for a vintage graphic tee under $30. I mostly wear baggy jeans and chunky sneakers. What's out there and how would I style it?"

**Step 1:**
The agent calls search_listings("vintage graphic tee", size=None, max_price=30) and receives matching listings.

**Step 2:**
The first matching listing is selected and stored as session["selected_item"]. The agent calls suggest_outfit(selected_item, wardrobe).

**Step 3:**
suggest_outfit returns a styling recommendation. The result is stored as session["outfit_suggestion"]. The agent then calls create_fit_card(outfit_suggestion, selected_item).

**Final output to user:**
The user receives the selected listing, an outfit recommendation, and a social-media style fit card describing the completed outfit.
